# Driftly — desktop app

Cross-platform Electron app: synthetic activity generator + activity benchmark.
Local-only, no telemetry. See the repository root `PLAN.md` for the full architecture.

## Develop

```bash
npm install
npm start        # launches Driftly (simulation mode if native modules are absent)
npm run check    # syntax-check all main-process modules
```

The optional native dependencies enable **real** input control and **global** input
monitoring:

- `@nut-tree-fork/nut-js` — drives the real mouse/keyboard;
- `uiohook-napi` — observes all real + synthetic input system-wide.

If they can't be installed/built for your platform, Driftly automatically runs in
**simulation mode**: the entire UI, scheduler, scoring, charts and CSV/JSON export still
work — only real-user monitoring and real input injection are unavailable.

## Build installers

```bash
npm run dist          # current OS
npm run dist:win      # Windows  (NSIS .exe)
npm run dist:mac      # macOS    (.dmg)
npm run dist:linux    # Linux    (AppImage)
```

Artifacts land in `release/`. Upload them to GitHub Releases — the website download
buttons point at `…/releases/latest`.

## Source map

| File | Role |
|------|------|
| `src/main/index.js`        | App lifecycle, tray, window, IPC, engine reconciliation |
| `src/main/store.js`        | Local config + metrics persistence (JSON, no network) |
| `src/main/input-backend.js`| Real (nut-js) or simulation input backend |
| `src/main/generator.js`    | Humanized synthetic-activity engine |
| `src/main/monitor.js`      | Global input monitor; tags synthetic vs real |
| `src/main/metrics.js`      | Per-minute scoring; shadow vs passive; export |
| `src/main/scheduler.js`    | Weekdays + time ranges → enable/disable generator |
| `src/preload/preload.js`   | Safe contextBridge API |
| `src/renderer/`            | UI (HTML/CSS/JS, dependency-free canvas charts) |
| `build/`                   | Icons + `make-icons.js` generator |

## Icons
